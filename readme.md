# AI Чатбот з Gemini та LangGraph

## Вибраний варіант та предметна область

Варіант 7. Менеджер задач «TaskBot»
Тематика та контекст задачі:
Веб-застосунок для управління задачами через діалог: створення,
редагування, відстеження статусу. Інтерфейс має бути чистим та
продуктивним.

Предметна область охоплює управління персональними завданнями (To-Do List) з категоризацією по статусам та пріоритетам, а також взаємодію з AI-асистентом для отримання інформаційної підтримки та автоматизації робочих процесів.

## Інструкція з запуску

### Передумови

- Python 3.10 або вище
- Активне підключення до інтернету для взаємодії з Google Gemini API
- API ключ Google (GOOGLE_API_KEY)

### Кроки встановлення

1. Клонуйте репозиторій або отримайте файли проекту.

2. Встановіть залежності в віртуальному середовищі:

```bash
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

3. Налаштуйте секретні дані. Створіть файл `.streamlit/secrets.toml` у кореневій папці проекту:

```toml
GOOGLE_API_KEY = "ваш_gemini_api_ключ_тут"
```

4. Запустіть додаток:

```bash
streamlit run app.py
```

Додаток буде доступний за адресою http://localhost:8501 у вашому браузері.

## Опис реалізованих функцій та UI-компонентів

### Режими роботи

#### 1. Звичайний режим чату (режим за замовчуванням)

- Опис: Прямий діалог з моделлю Gemini без викликання інструментів.
- Особливості:
  - Streaming-генерація відповідей з покроковим виводом у реальному часі
  - Контроль температури (0.0 до 1.0) для налаштування творчості та детермінованості відповідей
  - Простий та стабільний режим для базових запитів
  - Максимальна довжина відповіді обмежена 2048 токенами

#### 2. Режим агента з інструментами

- Опис: Розширений режим з можливістю автоматичного виклику спеціалізованих інструментів.
- Доступні інструменти:
  - current_datetime() — повернення поточної дати, часу та дня тижня у форматі UTC+3
  - create_task(title, priority) — створення нового завдання з заголовком та рівнем пріоритету
  - set_task_done(task_id) — позначення завдання як виконаного за його унікальним ідентифікатором
  - list_open_tasks() — отримання переліку всіх активних завдань
- Особливості:
  - Автоматичне розпізнавання намірів користувача та вибір потрібних інструментів
  - Інтеграція зі структурою завдань у інтерфейсі додатка
  - Debug-режим для відображення деталей викликів інструментів та результатів їх виконання
  - Розширене логування для діагностики та контролю процесу обробки

### Управління завданнями

#### Структура завдання

Кожне завдання містить такі поля:
- id: унікальний числовий ідентифікатор, який генерується автоматично
- title: текстовий опис змісту завдання
- status: поточний статус завдання (open, in_progress, done)
- priority: рівень пріоритету (High, Medium, Low)

#### Функції управління

- Швидке додавання: форма в бічній панелі для миттєвого створення нових завдань
- Фільтрація: фільтри за статусом та пріоритетом для зберігання релевантного переліку
- Редагування: можливість змінити статус та пріоритет безпосередньо в інтерфейсі завдання
- Синхронізація: автоматичне оновлення стану завдань при взаємодії з агентом
- Статистика: відображення кількості виконаних, активних та завдань у процесі

### Компоненти користувацького інтерфейсу

#### Бічна панель

- Режим: перемикач для вибору між звичайним чатом та режимом агента
- Інформація про модель: відображення вибраної версії Gemini (gemini-3.1-flash-lite)
- Температура: слайдер для контролю параметра моделі у звичайному режимі
- Debug агента: чекбокс для відображення технічних деталей викликів інструментів
- Фільтри завдань: селектори за статусом та пріоритетом
- Статистика: метрики виконаних, відкритих та активних завдань
- Швидке додавання завдання: форма для створення завдань без відкриття основного діалогу
- Доступні інструменти: розворотне меню з описом та параметрами інструментів агента

#### Основна область

- Чат: відображення повної історії діалогу з розрізненням повідомлень користувача та асистента
- Поле введення: текстове поле для введення повідомлень з кнопкою надсилання
- Завдання: таблична область з можливістю редагування параметрів кожного завдання
- Kanban-дошка: трьохколонна дошка для управління статусами (Відкриті, В процесі, Виконані)

### Функції обробки помилок та логування

- Розширене логування: поетапне відстеження роботи агента з виводом інформації у користувацький інтерфейс
- Обробка виключень: перехоплення та інформативний вивід помилок API та локальної обробки
- Перевірка результатів: валідація отриманих від агента даних перед виводом користувачеві
- Debug-інформація: можливість відображення технічних деталей викликів інструментів для діагностики

## Архітектура та технічні деталі

### Модель та параметри

- Версія: Google Gemini 3.1 Flash Lite
- Оптимізація: спеціально розроблена для швидкої обробки з мінімальною затримкою, що робить її придатною для інтерактивних додатків
- Температура у звичайному режимі: контролюється користувачем через слайдер
- Температура в режимі агента: встановлена на 0.0 для детермінованого та надійного вибору інструментів

### Архітектура LangGraph агента

Агент побудований з використанням LangGraph та складається з трьох основних вузлів графа:

1. agent_node — вузол, який обробляє вхідні повідомлення, взаємодіє з мовною моделлю та визначає, які інструменти необхідно викликати для виконання завдання

2. ToolNode — вбудований вузол LangGraph для виконання викликаних інструментів та отримання їх результатів

3. task_manager_node — спеціалізований вузол для обробки результатів інструментів, пов'язаних із завданнями, та синхронізації стану локальної системи завдань з результатами

Граф використовує in-memory checkpointer для збереження стану між викликами, що дозволяє підтримувати контекст діалогу.

## Конфігурація та налаштування

### Змінення версії моделі

Для використання іншої версії Gemini, відредагуйте константу MODEL_NAME у файлі agent.py:

```python
MODEL_NAME = "gemini-3.1-flash-lite"  # або інша доступна модель Gemini
```

### Налаштування системної інструкції агента

Для змінення поведінки та мовних характеристик агента, відредагуйте змінну SYSTEM_PROMPT у файлі agent.py:

```python
SYSTEM_PROMPT = SystemMessage(
    content=(
        "Ваша кастомна системна інструкція..."
    )
)
```

### Налаштування параметрів UI

Основні налаштування Streamlit сторінки можна змінити в блоці налаштування сторінки app.py, включаючи заголовок, піктограму та лайаут.

## Рекомендації щодо використання

- API ключ Google повинен зберігатися в захищений спосіб через файл .streamlit/secrets.toml, а не безпосередньо в коді проекту.

- В режимі агента рекомендується вмикати debug-режим під час розробки та тестування для контролю та аналізу викликів інструментів.

- Максимальна довжина вихідних токенів встановлена на 2048 для звичайного режиму відповідно до требів оптимізації затримки відповіді.

- При розробці нових інструментів для агента переконайтеся, що вони повертають структуровані дані у форматі JSON для коректної обробки task_manager_node.

- Для оптимальної роботи агента рекомендується вводити запити українською мовою, оскільки система інструкцій оптимізована для цієї мови.
